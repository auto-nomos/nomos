Every authorize, every step-up, every revocation lands in `audit_events`. Each row
hashes the previous row's hash plus its own canonicalized payload ‚Äî a Merkle list
that's tamper-evident from any signed root onward.

## The row shape

```jsonc
{
  "eventId": "evt_01J7K‚Ä¶",
  "ts": "2026-05-23T14:12:31.412Z",
  "decision": "allow",
  "command": "/github/issue/list",
  "resource": { "provider": "github", "owner": "acme", "repo": "app" },
  "context": { "now": {...}, "cosigner": false, "envelope_active": true },
  "principal": "did:key:z6Mk‚Ä¶",
  "agentName": "triage-bot",
  "swarmId": null,
  "chainDepth": 0,
  "parentReceiptId": null,
  "prevHash": "sha256:abc‚Ä¶",
  "hash": "sha256:def‚Ä¶"
}
```

`prevHash` is the row above. The chain is canonicalized in JSON Canonicalization
Scheme (JCS) before hashing, so two equivalent JSONs hash identically.

## Signed roots

Once a day, the broker signs the latest hash with an Ed25519 key (`AUDIT_SIGN_KEY`)
and stores it in `audit_roots`. The same payloads are mirrored as Parquet to R2 with
a 7-year lifecycle.

To prove an event existed at a point in time:

1. Note the event's `eventId` + `hash`.
2. Verify the row by re-canonicalizing + re-hashing.
3. Walk forward to the next signed root; confirm the hash chain.
4. Verify the signed root with `AUDIT_VERIFY_KEY` (the public side, posted in the
   dashboard's <Link href="/security">Security</Link> page).

## Drill-in

Click a row ‚Üí **Decision detail** drawer:

- Header: Event ID, Time, Command, Decision (`allow` / `deny` / `step_up`).
- **Record** and **Payload** ‚Äî the full canonicalized JSON (collapsible).
- Hash + prevHash (clickable, jumps to neighbor).
- For step-ups: linked cosigner UCAN cid + approver passkey credential id.
- **Verify offline** ‚Äî the drawer prints the exact command to re-prove this one row
  on your own machine:

  ```bash
  npx @auto-nomos/audit-verify <eventId>
  ```

- **Download proof** ‚Äî JSON bundle for the same offline check via the
  <Link href="/docs/operate/audit-verify-cli">audit-verify CLI</Link>.

## Connect

<Steps>
  <Step n="1" title="Initiate from the dashboard">
    <Link href="/app/connections">Connections</Link> → **Connect** → **GitHub**. The
    button opens GitHub's OAuth consent screen in a new tab.
  </Step>
  <Step n="2" title="Pick the right account / org">
    GitHub shows account picker. Pick the org you want Nomos to see. Per-org consent
    matters: a personal-account consent does *not* grant Nomos access to your work org.

    <Shot src="github-org-picker.png" alt="GitHub OAuth consent showing org picker" caption="Pick the org carefully. You can re-connect later to add more, but each org is its own consent." />
  </Step>
  <Step n="3" title="Approve">
    Nomos requests `repo`, `read:user`, `read:org`. These are the *ceiling*. Your
    Cedar policy then narrows what an agent can actually do.
  </Step>
</Steps>

## Starter policies

- **`github:read-only`** — `repo/get_file`, `repo/list_files`, `issue/*` read, `pr/list`. No writes.
- **`github:safe-default`** — all of read-only + `issue/create`, `issue/update`, `pr/create`.
  PR merge, file put, release create require step-up.
- **`github:org-pinned`** — same as safe-default but constrained to a single org.
- **`github:repo-pinned`** — narrowed to one repo. Useful for codegen agents.

Pick one in <Link href="/app/policies">Policies</Link> → **From template** → GitHub.

## Cedar fragment

```cedar
permit (
  principal,
  action in [Action::"/github/issue/list", Action::"/github/issue/get",
             Action::"/github/repo/get_file", Action::"/github/repo/list_files"],
  resource
) when {
  resource.owner == "acme" && resource.repo == "app"
};

permit (
  principal,
  action == Action::"/github/pr/create",
  resource
) when {
  resource.owner == "acme" && resource.repo == "app"
  && context.cosigner == true
};
```

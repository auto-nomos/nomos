Claude Desktop reads MCP servers from a single config file. Drop the Nomos block in,
restart Claude, and every Nomos-connected SaaS becomes a callable tool.

<Prereqs items={[
  "Claude Desktop 1.0+ (macOS, Windows, or Linux).",
  "A Nomos App with an API key (Get started → Register your first App).",
  "At least one Connection.",
]} />

## Where the config lives

<table>
  <thead><tr><th>Platform</th><th>Path</th></tr></thead>
  <tbody>
    <tr><td>macOS</td><td>`~/Library/Application Support/Claude/claude_desktop_config.json`</td></tr>
    <tr><td>Windows</td><td>`%APPDATA%\Claude\claude_desktop_config.json`</td></tr>
    <tr><td>Linux</td><td>`~/.config/Claude/claude_desktop_config.json`</td></tr>
  </tbody>
</table>

<Callout tone="info" title="File doesn't exist yet?">
  Create it. Claude Desktop won't auto-create the file but it reads it on every launch
  if it's there.
</Callout>

## The config block

```jsonc
{
  "mcpServers": {
    "nomos": {
      "command": "npx",
      "args": ["-y", "@auto-nomos/mcp-server@latest"],
      "env": {
        "NOMOS_CONTROL_URL": "https://control.auto-nomos.com",
        "NOMOS_API_KEY": "nk_live_…",
        "NOMOS_PDP_URL": "https://pdp.auto-nomos.com"
      }
    }
  }
}
```

If you already have other MCP servers, merge under the existing `mcpServers` object.

## Step-by-step

<Steps>
  <Step n="1" title="Open the config file">
    On macOS:
    ```bash
    open -a "TextEdit" ~/Library/Application\ Support/Claude/claude_desktop_config.json
    ```
    On Windows: open the file with Notepad. On Linux: `xdg-open ~/.config/Claude/claude_desktop_config.json`.
  </Step>
  <Step n="2" title="Paste the Nomos block">
    Drop the JSON above into the file. If the file already had `mcpServers`, add the
    `nomos` key alongside the existing ones — don't replace. Save.
  </Step>
  <Step n="3" title="Restart Claude Desktop">
    Quit Claude (cmd-Q / right-click tray → Quit). Re-open. On launch, Claude spawns
    the MCP server subprocess and registers its tools.
  </Step>
  <Step n="4" title="Verify in the tools picker">
    In a chat, click the wrench icon. You'll see Nomos-provided tools like
    `github_issue_list`, `slack_message_post`, `linear_issue_create`. The list reflects
    your active Connections + your App's policy.

    <Shot src="claude-desktop-tools-picker.png" alt="Claude Desktop tools picker showing Nomos tools" caption="One tool per allowed command, scoped by your App's policy." />
  </Step>
  <Step n="5" title="Ask Claude something that needs a tool">
    *"Read the README of acme/app."* Claude picks `github_file_get`, the MCP server
    asks Nomos for a UCAN, the PDP allows, GitHub returns the file. Claude reads it.

    <Shot src="claude-desktop-tool-call.png" alt="Claude Desktop showing a github_file_get tool call result" caption="Claude shows the tool name + arguments; the result is inlined in the response." />
  </Step>
</Steps>

<Verify>
  - Claude Desktop's tools picker lists Nomos tools (refresh after restart if not).
  - A test call ("list my GitHub repos") returns real data.
  - `/app/audit` shows one row per tool call.
</Verify>

## Approval flow on macOS

Step-ups deliver via:

- **Web push** to your dashboard browser (if open).
- **System notification** if you've enrolled the macOS notification permission.
- **Email** if neither of the above are wired.
- **Telegram** if you've set up the Telegram bot.

You tap, your passkey signs, Claude Desktop pauses for ~3 seconds while the cosigner
UCAN lands, then the tool returns the result.
